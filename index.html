git 命令学习

git工作流程

1, 工作区的文件修改之后 需要 通过 add命令 添加到 暂存区stage (版本库的第一个阶段)

2, 暂存区的修改 通过 commit 命令 把暂存区的所有内容提交到当前分支。

3个步骤

  1. git add .把所有文件放入暂存区；
  2. git commit把所有文件从暂存区提交进本地仓库；
  3. git push把所有文件从本地仓库推送进远程仓库。

4个区

  • 工作区(Working Area)
  • 暂存区(Stage)
  • 本地仓库(Local Repository)
  • 远程仓库(Remote Repository)

5种状态

  • 未修改(Origin)
  • 已修改(Modified)
  • 已暂存(Staged)
  • 已提交(Committed)
  • 已推送(Pushed)

git常用命令整理

1 ,添加修改到暂存区

  • git add 文件名
  • git add . //添加全部修改文件到暂存区

2, 查看历史记录(git log)

  • git log –pretty=oneline(log一行显示)

3, 版本回退

首先，Git必须知道当前版本是哪个版本，在Git中，用HEAD表示当前版本，也就是最新的提交3628164...882e1e0（注意我的提交ID和你的肯定不一样），上一个版本就是HEAD^，上上一个版本就是HEAD^^，当然往上100个版本写100个^比较容易数不过来，所以写成HEAD~100。

Git的版本回退速度非常快，因为Git在内部有个指向当前版本的HEAD指针，当你回退版本的时候，Git仅仅是把HEAD从指向指定版本：然后顺便把工作区的文件更新了。所以你让HEAD指向哪个版本号，你就把当前版本定位在哪。

  • 回退到指定的 commitid 阶段 版本 $ git reset --hard commit_id

  • 查看不同阶段的commitid 命令 git reflog 查看 版本切换记录 可以查看commitid

  • HEAD指向的版本就是当前版本，因此，Git允许我们在版本的历史之间穿梭，使用命令git reset --hard commit_id。

  • 穿梭前，用git log可以查看提交历史，以便确定要回退到哪个版本。

  • git log --pretty=oneline --abbrev-commit查看commit记录 一行显示

  • 要重返未来，用git reflog查看命令历史，以便确定要回到未来的哪个版本。

4，查看变化

  • git status查看工作区和暂存区的差异文件
  • git diff (file) 已修改，未暂存 查看具体文件的(没指定文件名就是全部) 变化内容(不识别中文) git diff这个命令只检查我们的工作区和暂存区之间的差异
  • git diff --cached已暂存，未提 查看看到暂存区和本地仓库之间的差异
  • git diff master origin/master已提交，未推送 查看 本地仓库和远程仓库的差异 master就是你的本地仓库，而origin/master就是你的远程仓库，master是主分支的意思，因为我们都在主分支上工作，所以这里两边都是master，而origin就代表远程。
  • cat file查看 指定文件的内容
  • git diff HEAD --(file) 查看工作区和版本库里面最新版本的区别git diff HEAD --看所有文件
  • git branch查看当前所在分支
  • git log --graph查看到分支合并图。
  • git shortlog <-s -n>查看commit记录(分帐号) -s 参数省略每次 commit 的注释，仅仅返回一个简单的统计。-n 参数按照 commit 数量从多到少的顺利对用户进行排序

5，撤销修改

  • git checkout -- file撤销工作区的修改（commit到暂存区之前）//不指定文件名无效
  • git reset (HEAD) (file) 已经添加到暂存区中，撤销暂存区的修改回退到工作区。HEAD表示最新的版本。git reset清空暂存区的内容
  • $ git reset --hard commit_id已经提交到版本库中，回退到指定commitid的版本
  • git reset --hard 未提交都可进行撤销所有修改

6，删除文件

  • 本地删除，版本库中有 删除版本库 git rm filt 然后git commit -m "desc"提交
  • 错删本地文件，从版本库中还原（撤销修改）$ git checkout -- file

7 远程分支管理

  • $ git clone (-b branch) https地址 (fileName) 克隆远程分支到本地，默认克隆master分支 -b克隆指定分支fileName克隆下来的文件夹名字

  • git checkout -b <name> <orign_name>创建+切换到分支(-b)<orign_name>表示从哪个分支创建新分支，默认从当前分支创建 git checkout命令加上-b参数表示创建并切换，相当于以下两条命令

  • $ git branch -a查看所有远程分支

  • $ git branch --no-merged查看尚未合并到当前分支的branch

  • $ git branch --merged查看已经合并的分支

  • $ git branch -d <name>删除指定分支

  • $ git branch -D <name>强行删除指定分支

  • $ git push origin --delete <branchName>删除远程分支

  • git push origin :<branchName>删除远程分支

  • $ git branch --set-upstream-to=origin/<branch> <local_branch>将远程分支和本地分支关联

  • git merge <name>命令用于合并指定分支到当前分支，Fast-forward表示“快进模式”，也就是直接把master指向dev的当前提交，所以合并速度非常快

  • git merge --no-ff -m "desc" <name> --no-ff参数表示禁用Fast forward

  • $ git pull <origin branch>把远程branch的代码更新到本地分支(默认更新本地分支对应的远程分支)

    (保存工作空间)保留当前分支的修改内容 临时去修改其他分支 再回到之前分支的修改

    • (保存工作空间)$ git stash把当前工作现场(有变化)“储藏”起来，等以后恢复现场后继续工作
    • (查看工作空间列表)$ git stash list 查看当前分支保存的工作空间stash列表
    • (恢复工作空间)$ git stash pop恢复的同时把stash内容也删了
    • (恢复工作空间)$ git stash apply (stash@{0})恢复(到指定的)stash内容
    • (删除工作空间)$ git stash drop删除指定stash内容

  • 查看远程库的信息$ git remote

  • git remote -v显示更详细的信息(可以抓取和推送的origin的地址。如果没有推送权限，就看不到push的地址)

推送分支

  • 本地新建的分支如果不推送到远程，对其他人就是不可见的；
  • 从本地推送分支，使用git push origin branch-name，如果推送失败，先用git pull抓取远程的新提交
  • 在本地创建和远程分支对应的分支，使用git checkout -b branch-name origin/branch-name，本地和远程分支的名称最好一致；
  • 建立本地分支和远程分支的关联，使用git branch --set-upstream <branch-name> origin/<branch-name>；
  • 从远程抓取分支，使用git pull，如果有冲突，要先处理冲突。

标签管理

  • $ git tag <name>在当前分支 新建标签
  • $ git tag <name> commit_id为指定commit_id 新建标签
  • $ git tag查看所有标签
  • git show <tagname>查看标签信息
  • 命令git tag <name>用于新建一个标签，默认为HEAD，也可以指定一个commit id；
  • git tag -a <tagname> -m "blablabla..."可以指定标签信息；
  • git tag -s <tagname> -m "blablabla..."可以用PGP签名标签；
  • 命令git push origin <tagname>可以推送一个本地标签；
  • 命令git push origin --tags可以推送全部未推送过的本地标签；
  • 命令git tag -d <tagname>可以删除一个本地标签；
  • 命令git push origin :refs/tags/<tagname>可以删除一个远程标签。
  • git push origin --delete tag <tagname>删除远程tag
